# Improve Strategy Selector UX

## 背景

策略页面当前使用两个 selectbox：先选策略大类，再选策略。Streamlit 在切换大类时会刷新页面，并且子策略默认落到该大类第一个策略，用户还没明确选择子策略就看到第一个策略的详情，体验容易误解。

## 需求

- 策略选择改成更明确的二级菜单体验。
- 切换策略大类后，不自动选中该大类第一个子策略。
- 如果当前已使用策略属于所选大类，可以默认选中当前策略。
- 如果当前已使用策略不属于所选大类，提示用户选择子策略。
- 不改变“一键使用”“复制策略”“临时应用”“另存为”的业务逻辑。

## 验收

- 切换大类后页面刷新，但不会自动打开第一个子策略编辑表单。
- 用户必须明确选择子策略后才展示策略详情和编辑表单。
- Python 语法检查通过。
