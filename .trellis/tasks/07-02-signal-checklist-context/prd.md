# Improve Signal Checklist Strategy Context

## 背景

信号页展示的是数据库历史信号的检查项。用户切换策略后，如果机器人还没有生成新信号，页面仍显示旧策略的检查项，容易误以为检查项是固定的或没有跟随当前策略变化。

## 需求

- 信号详情展示该信号生成时使用的策略名称、大类、方向模式和周期。
- 新增“按当前策略刷新信号”能力，只重算并写入信号，不执行风控和下单。
- 信号页说明检查项来自信号生成时的策略快照，不是实时随页面选择变化。
- 保持自动机器人原逻辑不变。

## 验收

- 切换策略后，用户可以手动刷新信号检查项。
- 刷新信号不会下单。
- 历史信号能看出当时使用的是哪个策略。
- Python 语法检查通过。
