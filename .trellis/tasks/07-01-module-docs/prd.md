# Document Module Responsibilities

## 背景

项目已经包含技术文档和功能文档，但模块职责、核心类边界、交易调用链路还不够精确。后续可能更换 AI 模型继续开发，需要更清晰的项目交接入口。

## 需求

- 在技术文档中补充项目结构、模块职责、核心类说明和关键交易链路。
- 在 `AGENTS.md` 中提醒新 AI 开发前阅读模块与核心类章节。
- 在核心类入口增加简短中文 docstring，说明业务职责和边界。
- 注释保持克制，不增加重复代码表面行为的注释。

## 验收

- 文档能帮助新 AI 快速定位信号、风控、执行、退出、数据库相关代码。
- Python 文件语法检查通过。
- 不改动业务逻辑。
