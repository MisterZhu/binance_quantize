# Document Strategy Logic

## 背景

用户依赖文档理解开单策略，并希望后续通过修改文档表达优化需求，再由 AI 对照执行。因此策略文档必须和代码逻辑完全一致，不能只写概念描述。

## 需求

- 用中文补充策略入场逻辑文档。
- 文档必须逐项对应 `core/strategy/ema_structure.py`、`support_resistance.py` 和 `indicators.py` 的实际判断。
- 覆盖三类策略：趋势突破、趋势回调、日内波动回归。
- 说明启用条件、评分规则、方向限制、做空限制、止损止盈和 RR 计算。
- 保留“未经过回测验证”的风险提示。

## 验收

- 用户能从文档看到每个 UI 检查项背后的实际公式。
- 文档和代码中的条件 key 保持一致。
- 不改业务逻辑。
